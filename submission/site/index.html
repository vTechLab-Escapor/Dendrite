The problem

AI chat is a straight line. Thinking isn't.

Every reply pushes the last one out of view; chasing a tangent means abandoning your main thread. And the compute lives in the cloud — so privacy, offline use, and long-run cost all suffer.

😮‍💨 Linear chat

  • Follow a side-thread → the main line scrolls into oblivion
  • Compare two answers → copy-paste between sessions
  • A rich exploration ends up an unstructured transcript
  • The model runs in the cloud — your data leaves the device

🌿 Dendrite

  • Highlight a phrase → spawn a branch; context is inherited exactly
  • A live mind-map of the whole tree — nothing is ever lost
  • One-click export to structured, traceable Markdown
  • Default to a local Qwen model (MNN) — offline, private, zero-cost
What it does

One question. Many directions. Nothing lost.

A complete, local-first product — not a prototype. Verified end-to-end on a real Android device.

🌳

Non-linear branching

Select any text in any reply and branch off it. Each branch carries its selection as context, so the model knows exactly what you're drilling into.

🗺️

Live mind-map

The whole conversation as an interactive tree. Pan, zoom, tap a node to jump. The active lineage is highlighted; key results are starred.

📱

On-device + cloud

A local Qwen model via MNN (Arm SME2-ready) for offline, private inference — or switch to the cloud per chat, even per branch.

🌿

arXiv / GitHub analysis

Paste a paper or repo link — Dendrite fetches it and auto-grows a multi-branch analysis tree: contribution, method, experiments, limitations…

🧠

Lineage-aware context

The model only sees the ancestor path of the current node — one recursive SQL CTE, not the whole tree. Focused context, lower token cost.

⚡

True real-time streaming

Token-by-token SSE via dart:io, deliberately bypassing the Android buffering that stalls package:http.

🔒

Local-first & secure

All history in on-device SQLite (Drift). Cloud API keys live in the platform secure store — never in plaintext.

🔀

Multi-provider

One engine, three wire dialects: OpenAI-compatible, Anthropic, and Gemini. Bring your own key; nothing is proxied.

📦

Export & reuse

The entire tree → structured Markdown (branches become nested headings), ready for slides, docs, or review.

WITH RECURSIVE lineage(id) AS (
  SELECT id FROM messages WHERE id = ?1
  UNION ALL
  SELECT m.parent_id FROM messages m
  JOIN lineage l ON m.id = l.id
  WHERE m.parent_id IS NOT NULL
)
SELECT m.* FROM messages m JOIN lineage l ON m.id = l.id
ORDER BY m.created_at ASC, m.rowid ASC;

self.close_connection = True          # close-delimited; keep-alive stalls dart:io
self.send_header("Connection", "close")

    self.wfile.flush()

Engineering

Real challenges & how they were solved

A selection of the bugs and constraints solved while taking this from prototype to a verified on-device build.

1 Release APK had no network — debug was fine.

Root cause: the INTERNET permission lived only in the debug manifest, which isn't merged into release. Fix: declare it in the main manifest; verified in the merged release manifest.

2 On-device SSE stalled — server returned 200, app got no tokens.

Root cause: keep-alive with no length/chunking — dart:io waits for a body boundary forever. Fix: close-delimited streaming; verified in ~10s with a host-side Dart probe that replicates the app's client.

3 App crashed during multi-branch analysis.

Root cause: feeding partial markdown to flutter_markdown trips an internal _inlines.isEmpty assertion. Fix: render the actively-streaming node as plain text, switch to rich markdown only once complete.

4 Crash after a screen closed (_positions.isNotEmpty).

Root cause: a delayed scroll callback fired after dispose. Fix: re-check mounted && hasClients inside the callback.

5 Run it all end-to-end on a real device (WSA).

Solution: derive the host gateway from adb shell ip route; a scoped network_security_config allows cleartext only to that local IP (prod stays HTTPS); a one-command runner orchestrates WSA + MNN + the integration test.

6 GitHub API rate-limit (shared exit IP via WSA).

Fix: optional Bearer-token support, and the single-file path uses the raw CDN — which has no API limit.

7 A secret slipped into git history.

Root cause: a VolcEngine presigned-URL credential in a data-analysis artifact. Fix: redacted across all history with git filter-branch; push protection verified clean.

8 Producing a shot-matched demo video, automated.

Constraints: screenrecord ≤180s, variable build time, on-screen overlays. Fix: an offline mock builds the tree instantly to fit every UI beat in one take; clips are cut to each line's dwell and assembled with TTS narration + burned subtitles.
