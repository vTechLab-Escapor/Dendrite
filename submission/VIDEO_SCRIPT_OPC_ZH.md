## 🎯 核心叙事：Dendrite 评审 Dendrite（自评闭环）

> 真实任务 = 「对 Dendrite 这个 Flutter 仓库做一次完整架构评审，把结论沉淀成可追溯的知识，再一键生成给评委/管理层的汇报 PPT」。
>
> 用 **Dendrite** 把这次自评散开成一棵知识树（16 节点 / 7 分支 / 6 条收藏结论，见 `submission/Dendrite-Branch-Map.md`）→ 一键 **导出 Markdown**（`submission/Dendrite-PPT-input.md`）→ 喂给 **办公小浣熊**的 PPT 技能 → 自动生成 `submission/Dendrite-Raccoon-PPT.pptx`。
>
> **关键钩子（结尾点破）：你现在看到的这份参赛 PPT，就是 Dendrite 对自己做架构评审、再交给小浣熊生成出来的。Dendrite 评审了 Dendrite。**
>
> 而且评审是**真**的——树里浮现的是仓库里真实存在的问题，不是套话：
> - 搜索注释写着 `FTS5`，实际只是 `content LIKE '%q%'`，无索引无排序（名实不符的技术债）。
> - Gemini 把 API Key 拼进 URL query，可能被代理 / 网关 / 日志留存（🔴 中危安全项）。
> - `chat_screen.dart` 约 2276 行，是职责过载的 god-widget。

**与评分维度的对应：**
- **真实场景 & 闭环验证（35%）** → 场景就是「我现在要交的这份 PPT」，且工具对自己做了可验证的真实评审；演示从评审一路跑到成品 PPT，端到端闭环、产物可见、问题可核对。
- **业务价值 & 可持续（30%）** → 把一次发散的代码评审沉淀成结构化、可追溯文档，可反复复用；本地优先 + 自带密钥，零额外成本长期运行。
- **工具整合 & 替换能力（25%）** → Dendrite（分支 / 思维导图 / 收藏 / 导出）+ 办公小浣熊（PPT 生成）多模块协同；且 Dendrite 引擎本身可在 GLM / NVIDIA / OpenAI / Anthropic / Gemini 间真实切换（镜头里现切一次）。
- **表达清晰（10%）** → 旁白按「痛点 → 方案 → 真实评审 → 闭环 → 价值」推进，字幕全程跟随。
