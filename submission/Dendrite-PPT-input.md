## 封面
- **标题：** Dendrite —— 用分支思考，别走直线
- **副标题：** AI 原生的非线性知识空间（Flutter · Android / iOS / Web）
- **一句话定位：** 用「可分支对话树 + 一键生成 PPT」解决大型代码仓库评审中的知识沉淀难题。

## 1. 场景与痛点（Why）
- **真实场景：** 开发团队评审 / 分析一个大型代码仓库——新项目评审、技术债评估、或迁移前准备。成员一边用聊天工具 / IDE 插件实时记录想法、摘录代码片段、标注风险点。
- **三个致命缺陷：**
  1. **多路径探索困境**：评审思路并行展开（架构、性能、安全同时推进），线性对话只能顺序记录，不同方向相互覆盖。
  2. **结论查找难题**：关键结论、Bug 根因、优化建议散落在冗长聊天记录里，事后难定位、难追溯。
  3. **汇报整理耗时**：评审结束后，论点与发现很难沉淀成可追溯、可复现的知识，手工整理汇报费时。

## 2. 解决方案总览（What）
- 一句话：**把讨论文本、代码、注释整合成一棵可追溯的树状对话。**
- 在任意一句话上**开新分支**，保留主线、并行记录思路；选中内容自动成为该分支上下文，互不污染。
- **思维导图**俯瞰整棵树，**收藏 + 全局搜索**沉淀关键节点，**一键导出 / 生成 PPT** 完成汇报。

## 3. 七大关键能力（What）
1. **非线性分支**：在任意节点开分支，多条思路并行而互不污染（本案例 16 节点产生 7 分支、沉淀 6 条结论）。
2. **本地安全存储**：对话 / 收藏 / 附件存本地 SQLite；API Key 仅存系统密钥库（iOS Keychain / Android Keystore）。
3. **多模型支持**：统一封装 **OpenAI / Anthropic / Gemini**，按需无缝切换。
4. **流式响应**：SSE 逐 token 实时呈现，交互顺滑。
5. **全局搜索与收藏**：快速定位关键节点、星标重要结论。
6. **PPT 生成接口**：把完整对话树序列化为 JSON / Markdown，交给 **raccoon-ppt-skill** 输出结构化 `*.pptx`。
7. **一键下载 / 分享**：自动本地保存并弹出分享面板，支持邮件、企业 IM 等渠道。

## 4–9. 实战案例：用 Dendrite 评审 Dendrite 自身仓库（每条分支单独成页）
> 统一版式：**选中上下文（代码片段）→ 提问 → 结论**。

- **案例 ①　多供应商 SSE 引擎**
  `AgentEngine` 用 `ApiDialect` 区分 openai / anthropic / gemini，按方言分派 `_endpoint / _buildHeaders / _buildBody / _extractDelta`；用 `dart:io` 原生 SSE 绕开 Android 缓冲，实现真正逐 token 流式。
- **案例 ②　安全风险**
  Gemini 把 key 放进 URL query，可能被代理 / 网关 / 日志留存（🔴 中危，建议改请求头携带并对日志脱敏）。
- **案例 ③　树与上下文**
  单张 `Messages` 表以 `parentId` 构成树；一条 `WITH RECURSIVE` CTE 取祖先链；模型上下文只取「当前节点→根」，控 token、分支互不污染。
- **案例 ④　技术债**
  搜索的注释写着 FTS5，实际是 `content LIKE '%q%'`——无索引、无排序，规模化会退化（建议补建真正的 FTS5，保留 LIKE 作降级）。
- **案例 ⑤　密钥存储**
  `flutter_secure_storage`（Keychain / Keystore）存 API Key；非敏感配置存 JSON；旧版明文密钥 `load()` 时自动迁移并从文件剔除。
- **案例 ⑥　一键成稿**
  `buildTreeMarkdown` 把整棵树（每条分支）序列化为嵌套标题、剥离 `<think>`、标注 ⭐ 收藏结论，交给 raccoon-ppt-skill。io / web 条件导出。

> 另有可维护性发现（可并入风险页）：`chat_screen.dart` 约 2276 行，建议拆分；`_extractDelta` 对解析异常静默 catch；`getChatSessions` 一次性全量读入内存。

## 10. 成果与价值（标注「预计」）
- **知识可追溯 / 可复现**：每条结论可定位到具体分支，完整保留「提问 → 探索 → 结论」路径；本案例 6 条收藏结论均可追溯到 16 个节点。
- **整理汇报提速**：传统手工整理约需 2–4 小时，本流程探索→导出→成稿一条龙，**预计**降低约 60% 整理时间。
- **零服务器 / 订阅成本**：本地优先 + 自带密钥，可个人长期使用。
- **可复用**：同一套「Dendrite 散开 → 导出 → 小浣熊成稿」流程适用于任意调研 / 汇报 / 评审场景。

## 11. 工具协同
- **Dendrite**（发散、结构化、导出）× **办公小浣熊 / raccoon-ppt-skill**（聚合、排版、成稿）= 从「想清楚」到「讲清楚」的完整闭环。
- 引擎本身可在 OpenAI / Anthropic / Gemini 间切换，按需选更强 / 更省的模型。
