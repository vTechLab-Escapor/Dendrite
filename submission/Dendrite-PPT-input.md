# Dendrite —— 办公小浣熊 PPT 生成输入稿

> 用法：把本文件**整段**粘进 办公小浣熊（office.xiaohuanxiong.com）的「一键生成 PPT」输入框，风格选 **商务 / 科技**，页数 **10–12 页**。这就是「Dendrite 散开知识树 → 导出 → 小浣熊成稿」闭环里喂给小浣熊的稿子。
> 提示词（可放在最前面）：`请基于以下结构化大纲，生成一份风格统一、可直接上台的参赛 PPT。每个二级标题成一页，要点精炼、配图建议照做。`

---

## 封面
- **标题：** Dendrite —— 用分支思考，别走直线
- **副标题：** 一个 AI 原生的非线性知识空间（Android / iOS / Web）
- **一句话定位：** 把 AI 对话从一条单向时间线，变成一棵可探索、可复用的知识树。

## 1. 场景与痛点（Why）
- **真实场景：** 用 AI 做深度调研 / 整理一份汇报或参赛 PPT 时，要来回追问、查资料、归纳。
- **线性对话的三个致命缺陷：**
  1. **主线被冲走**：想顺着一个岔路深挖，长长的滚动条把主线埋没，找不回来。
  2. **上下文复制粘贴**：要对比两个方向，只能开一堆新对话窗口，手动反复粘贴上下文。
  3. **思路无法沉淀**：聊完即散，关键结论散落在多个会话里，事后基本检索不回。
- **传统做法为什么不够：** ChatGPT 类产品是「信息流」，没有结构；笔记软件能存但不能「就地追问」；两者割裂。

## 2. 解决方案总览（What）
- 一句话：**每一段对话都是一棵树，不是一条流。**
- 选中任意一句回答 → **分支**，就这个点单独开一条探索线，选中内容自动成为该分支上下文，主线不受影响。
- **思维导图**俯瞰整棵树，当前路径发光，点节点一键跳回。
- **收藏 + 全局搜索 + 一键导出**：把一次性问答沉淀成可复用知识库。

## 3. 技术方案与实现（How）— 核心
- **树形数据结构：** 对话节点存于本地，父子关系构成树；模型上下文只取「当前节点→根」的祖先链（一条 `WITH RECURSIVE` SQL 解析），岔路便宜且不串味。
- **统一多供应商流式引擎：** 一套引擎抹平 NVIDIA / OpenAI / Anthropic / Gemini 的协议差异——协议适配层统一入口、统一事件流解析三种 SSE、响应式状态机驱动 UI 逐字流式。
- **交互式思维导图：** `InteractiveViewer` + `CustomPainter` 画三次贝塞尔连线，高亮当前祖先链。
- **隐私优先本地存储：** 所有对话只存本地 SQLite，API 密钥存设备安全区（Keychain/Keystore），不经服务器。
- **跨端一致：** Flutter 一套代码跑 Android / iOS / Web。

## 5. 成果与价值（What / 量化 · 标注「预计」）
- 整理一份项目汇报 PPT：传统要在仓库 / 聊天 / 文档间切换 **6–8** 个窗口、手动归纳大纲约 **1–2** 小时；本方案探索→导出→成稿一条龙，提纲自动成形，**预计**节省约 **60%** 整理时间。
- 探索零丢失：关键结论 **100% 可检索**（收藏 + 全局搜索 + 导出存档）。
- 运行成本：本地优先 + 自带密钥 → **0** 服务器 / 订阅成本，可个人长期使用。
- 可复用：同一套「Dendrite 散开 → 导出 → 小浣熊成稿」流程适用于任意调研 / 汇报 / 文档场景。

## 6. 工具协同（呼应评审「工具整合」）
- **Dendrite**（发散、结构化、导出）× **办公小浣熊**（聚合、排版、成稿）= 从「想清楚」到「讲清楚」的完整闭环。
- Dendrite 引擎本身可在多家大模型间切换，按需选更强 / 更省的模型。
