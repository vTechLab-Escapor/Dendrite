## 一、关键代码片段

### 1. 「焦点上下文」：单条递归 CTE 取祖先血脉
不把整棵树丢给模型，只取当前节点的**祖先路径**，毫秒级、低 token。
`lib/core/db/database.dart`
```sql
WITH RECURSIVE lineage(id) AS (
  SELECT id FROM messages WHERE id = ?1
  UNION ALL
  SELECT m.parent_id FROM messages m
  JOIN lineage l ON m.id = l.id
  WHERE m.parent_id IS NOT NULL
)
SELECT m.* FROM messages m
JOIN lineage l ON m.id = l.id
ORDER BY m.created_at ASC, m.rowid ASC
```

### 2. 端侧 Qwen：MNN 的 OpenAI 兼容本地服务（**连接关闭定界** SSE）
用 stdlib 暴露 `MNN.llm`，与云端走**完全相同**的客户端代码路径。关键：SSE 必须以「连接关闭」定界，否则 `dart:io` 客户端会一直等 body 边界而卡死。
`tool/mnn/server.py`
```python
self.close_connection = True          # close-delimited; keep-alive 会让 dart:io 卡住
self.send_header("Connection", "close")
...
self.wfile.write(b": connected\n\n")   # 立即给字节，先发头再生成
text = self.backend.generate(messages) # MNN.llm.create(cfg).load().response(...)
for piece in _chunk_text(text):
    self.wfile.write(_sse({"choices":[{"delta":{"content":piece}}]})); self.wfile.flush()
self.wfile.write(b"data: [DONE]\n\n")
```

### 6. 流式期间用纯文本渲染（规避 flutter_markdown 断言）
把**半截 Markdown** 喂给 `flutter_markdown` 会触发内部 `_inlines.isEmpty` 断言崩溃；解法是流式时渲染纯文本，**完成后再转富文本**。
`lib/features/chat/widgets/chat_screen.dart`
```dart
final isStreamingNode = _isStreaming &&
    _currentLineage.isNotEmpty && _currentLineage.last.id == node.id;
if (isStreamingNode) return Text(text);   // 完成后才走 MarkdownBody
```

## 二、技术挑战与解决方案（均为真实踩坑）

| # | 挑战 / 现象 | 根因 | 解决方案 |
|---|---|---|---|
| 1 | **Release 包完全无网络**（Debug 正常） | `INTERNET` 权限只在 debug manifest，发布构建不合并 | 把 `<uses-permission INTERNET>` 写入主 manifest；并入合并后的发布清单已核验 |
| 2 | **端侧 SSE 卡死**：服务端收到请求、应答 200，App 却抽不到 token | 服务端 `keep-alive` 且无 Content-Length/分块，`dart:io` 等不到 body 边界 | 改为**连接关闭定界**（`Connection: close`）；并写了**宿主端 dart 探针** `tool/mnn/sse_probe.dart` 复刻 App 客户端，10 秒验证而非 90 秒真机循环 |
| 3 | **多分支分析时 App 崩溃**（集成测试报 `_inlines.isEmpty`） | 流式过程中把不完整 Markdown（未闭合 `**`/`` ` ``）喂给 flutter_markdown | 流式期间渲染纯文本，节点完成后再 Markdown（见上） |
| 4 | 测试结束后偶发崩溃 `_positions.isNotEmpty` | `_scrollToBottom` 的延时回调在 widget 销毁后仍 `animateTo` | 回调内复检 `mounted && hasClients` 再滚动 |
| 5 | **WSA 真机端到端跑通** | 真机 → 宿主 MNN 的网络、明文 HTTP、网关 IP | `adb shell ip route` 推导宿主网关；`network_security_config` 仅对内网网关放行明文（生产仍强制 HTTPS）；`tool/wsa/run_e2e.sh` 一键编排 |
| 6 | GitHub API 限流（与 WSA 共用宿主出口 IP，60/hr 很快耗尽） | 未鉴权限额 + 共享 IP | 加 **Bearer Token** 支持（`IngestService→GithubFetcher`）；单文件分析改走 **raw CDN（无限额）** |
| 7 | 端侧自动化测试难驱动「分支」 | 原生侧划词选区无法被合成手势触发 | 用 `ingestUrl` **自动建分支**（无需选区）作为测试入口；`integration_test/` 真机全绿 |
| 8 | Qwen3 思考模式冗长、MNN `max_new_tokens` 不生效 | 该 MNN 构建忽略 `set_config/config` 的 token 上限；模型仅按 EOS 停 | 模型分级落地：以 **Qwen3-1.7B** 保证「完整作答不空答」，0.5B 仅做快速管线验证 |
| 9 | **`git push` 被密钥扫描拦截** | 历史某 commit 的 raccoon 数据分析产物里含 VolcEngine 预签名 URL 凭证 | `git filter-branch` 跨全history 红action（`X-Amz-Credential/Signature`→REDACTED），核验后再推送 |
| 10 | **演示视频自动化产出** | `screenrecord` 单文件 ≤180s；真机构建耗时波动；指针叠层/DEBUG 横幅；`ffmpeg -vf ass=` 绝对路径冒号解析失败 | 用**离线 mock** 让树秒级生成把所有 UI 镜头塞进一次录制；`debugShowCheckedModeBanner=false` + 关 `pointer_location`；`ass` 用**相对路径** + `cwd`；按 `dwell.json` **逐镜对齐**装配 `assemble_mobile_v2.py` |

## 三、项目补充信息

**技术栈**：Flutter（Android 优先，单代码库）· flutter_bloc/Cubit · Drift+SQLite（本地优先，密钥进 Keystore）· OpenAI/Anthropic/Gemini 三方言 SSE 引擎 · **MNN** 端侧 Qwen。

**模型**：端侧 **Qwen3-1.7B（MNN）** 已验证；云端 **Qwen**（DashScope/魔搭 OpenAI 兼容）按需协同；多模态路线 **Qwen3-VL**。**端、云两侧均为 Qwen**。

**已验证（真机 WSA / Android 13）**：
- 对话往返、arXiv《Attention Is All You Need》→ **12 节点/5 分支全答**、GitHub `micrograd` → **10 节点/4 分支全答**；
- 单元测试 13/13、`flutter analyze` 无错；`flutter build apk --release` 成功（~57MB）。

**一键复现**：
```bash
TEST=integration_test/ingest_e2e_test.dart \
MODEL_DIR=D:/mnn-models/Qwen3-1.7B-MNN \
tool/wsa/run_e2e.sh
```

**关于 Arm SME2（诚实声明）**：MNN 提供面向 **Arm SME2** 的加速后端；本次验证宿主 CPU 无 SME2（运行时探测为 0），故 SME2 加速**未实测启用**——架构已为 on-device Arm 推理就绪（同一 OpenAI 兼容契约），真机 SME2 实测为明确部署目标。详见《技术可行性说明》。

**仓库 / 材料**：源码 + 创意提案 / 技术可行性说明 / 演示视频脚本_分镜 / 最终演示视频（`submission/`）。
